## Getting Started

### For Users
Start with the main [README.md](../README.md) for installation and usage instructions.

### For Testers
1. Read [testing.md](testing.md)
2. Run automated tests: `pytest`
3. Follow manual testing checklist

### For Developers
1. Read [development.md](development.md)
2. Set up development environment
3. Review test suite
4. Start contributing

### For Project Stakeholders
Read [project_summary.md](project_summary.md) for project status and metrics.

## Keeping Documentation Updated

When making changes to the plugin:
1. Update relevant documentation sections
2. Add entries to CHANGELOG.md
3. Update examples if API changes
4. Review and update test scenarios
5. Update project_summary.md if major milestones are reached

## Documentation Standards

- Use clear, concise language
- Include code examples where appropriate
- Keep examples up-to-date with code
- Use proper Markdown formatting
- Link to related documentation
- Include version information where relevant

## Contributing to Documentation

Documentation improvements are always welcome! Please:
1. Follow existing formatting and style
2. Test any code examples
3. Check for spelling and grammar
4. Update the table of contents if needed
5. Submit changes via pull request
